基本使用

git init 创建仓库
一般情况下在一个项目的根目录里都会去执行这个命令 项目的版本控制仓库 千万别手一抖给删了

工作区 正常的项目目录我们称之为工作区

暂存区 代码版本变化信息临时存储的区域 代码修改之后需要先提交到暂存区

分支仓库 代码版本信息真正存储的位置 一般来说先到暂存区 存储到仓库
ls查看当前文件夹下面的目录 ls是linux命令

git add 文件名 将文件提交到暂存区
git commit -m将暂存区的内容提交到仓库内
